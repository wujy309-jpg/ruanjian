## B11：MinIO资源存储对接

### 1. MinIO配置信息

```yaml
minio:
  endpoint: http://localhost:9000
  bucket-name: java1229
  username: minioadmin
  password: minioadmin
  url-expiry: 604800  # 7天
```

### 2. FileUploadService使用方式

```java
@Autowired
private FileUploadService fileUploadService;

// 上传文件
Map<String, Object> result = fileUploadService.uploadFile(file, "resources");
String url = (String) result.get("url");
```

### 3. 生成资源存储流程

对于AI生成的文档、思维导图等较大内容：

```java
// 1. 生成内容
String content = "...AI生成的内容...";

// 2. 转换为文件并上传
MultipartFile file = convertToFile(content, "resource.md");
Map<String, Object> uploadResult = fileUploadService.uploadFile(file, "generated-resources");
String contentUrl = (String) uploadResult.get("url");

// 3. 存储URL到数据库
GeneratedResource resource = new GeneratedResource();
resource.setContentUrl(contentUrl);
resource.setContentJson(null); // 大内容不存JSON，只存URL
generatedResourceMapper.insert(resource);
```

### 4. 文件访问

通过已有 `/files/**` 接口访问MinIO中的文件。

### 5. 本地存储备用

如果MinIO未配置或不可用，系统会自动降级到本地文件存储：
- 存储路径：`./uploads/`
- 访问URL：`http://localhost:8080/files/{folder}/{filename}`

## 注意事项

1. **Mapper公开**：所有新增Mapper均为public接口，A模块可直接注入使用
2. **JSON字段处理**：`user_profile.dimensions` 和 `agent_message.metadata` 使用JacksonTypeHandler
3. **外键约束**：删除数据时需注意顺序（先删子表再删主表）
4. **MinIO降级**：MinIO不可用时自动降级到本地存储
